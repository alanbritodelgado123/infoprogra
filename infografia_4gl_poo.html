Lenguaje de IV Generación (4GL)

Definición: Los lenguajes de cuarta generación (4GL) son lenguajes de programación de alto nivel diseñados para reducir el esfuerzo de programación y el tiempo de desarrollo. Se enfocan en qué se quiere lograr, no en cómo hacerlo.

Características Principales

Integración con bases de datos

Productividad

Desarrollo rápido de aplicaciones

Visual

Interfaces gráficas de desarrollo

Soporte nativo para POO

Modular

Componentes reutilizables

Los 3 Pilares de la POO

Encapsulamiento

Oculta los detalles internos de un objeto y expone solo lo necesario. Protege los datos mediante modificadores de acceso.

private String nombre;
public String getNombre() {
  return nombre;
}

Herencia

Permite crear nuevas clases basadas en clases existentes, heredando sus atributos y métodos. Promueve la reutilización.

class Perro extends Animal {
  // Hereda de Animal
  void ladrar() { }
}

Polimorfismo

Permite que objetos de diferentes clases respondan al mismo mensaje de formas distintas. "Muchas formas".

Animal a = new Perro();
a.hacerSonido(); // Ladra
Animal b = new Gato();
b.hacerSonido(); // Maulla

La Clase: Plantilla del Objeto

Clase: Es una plantilla o molde que define la estructura (atributos) y comportamiento (métodos) que tendrán los objetos creados a partir de ella. Es el plano a partir del cual se construyen objetos.

Clase: Automovil
Atributos
- String marca - String modelo - int año - double velocidad
Métodos
+ void acelerar() + void frenar() + String getInfo()
instancia
Objeto: miAuto
marca = "Toyota" modelo = "Corolla" año = 2023 velocidad = 0.0

El Objeto y sus Componentes

Un objeto es una instancia de una clase con estado y comportamiento propio

OBJETO

Propiedades

Características o atributos que describen el estado del objeto

Datos
nombre = "Juan"
edad = 25
activo = true

Métodos

Acciones que el objeto puede realizar, definen su comportamiento

Acciones
caminar()
hablar()
calcularEdad()

Funciones

Operaciones que retornan un valor después de procesar datos

Retorno
getNombre()

Eventos

Respuestas a acciones del usuario o del sistema

Triggers
onClick()

onChange()

Ventajas de la POO

  • Reutilización de código mediante herencia
  • Modularidad y organización del código
  • Facilita el mantenimiento y actualizaciones
  • Abstracción de problemas complejos
  • Seguridad mediante encapsulamiento
  • Flexibilidad con polimorfismo

Ejemplos de 4GL

  • SQL - Consultas a bases de datos
  • Python - Programación de alto nivel
  • MATLAB - Cálculo matemático
  • R - Análisis estadístico
  • Visual Basic - Desarrollo RAD
  • PowerBuilder - Aplicaciones empresariales

Modificadores de Acceso

  • public Accesible desde cualquier lugar
  • private Solo dentro de la clase
  • protected Clase y subclases
  • default Dentro del paquete

Clase vs Objeto

  • Clase: Plantilla, molde, plano
  • Objeto: Instancia concreta de la clase
  • Una clase puede tener múltiples objetos
  • El objeto tiene valores específicos
  • La clase define la estructura
  • El objeto representa la realidad
